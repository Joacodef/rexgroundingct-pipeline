# Experiment 005: VoxTell v1.1 Second Baseline Verification (Full Validation Set)

**Date**: July 15, 2026
**Purpose**: Second verification of the official `ibrahimhamamci/voxtell_v1.1` weights, now evaluating on the complete set of 200 validation masks (released in July). Includes deep qualitative error analysis to identify the types of erroneous predictions generated by VoxTell.

**Execution**:
- Used a Bash Chunking Wrapper to mitigate massive GPU memory leaks in the `VoxTellPredictor`.
- Inference was chunked in batches of 10 volumes.
- `tile_step_size` was set to `0.75` (25% overlap) to accelerate inference.
- GPU Used: Physical GPU 1 (RTX PRO 6000 Blackwell, 96GB).

**Results**:
- **Overall Dice**: 0.0975
- **Hit Rate**: 24.14%
- **By Size**:
  - Small (<1k): Dice 7.34%
  - Medium (1k-10k): Dice 7.08%
  - Large (>10k): Dice 13.91%

**Analysis & Bug Suspicions**:
The Dice score dropped precipitously compared to the >0.20 score on the official leaderboard. Two main hypotheses for this performance gap:
1. **Aggressive Tile Step Size**: We used `tile_step_size = 0.75` (25% overlap). This likely destroyed context at the patch boundaries, obliterating the performance on small and medium findings.
2. **Affine / Reorientation Bug**: `voxtell_inference.py` performs a complex `(F, Z, Y, X) -> RAS -> Original Space` transformation to fix a metadata bug in the Ground Truth. It is possible the official evaluator evaluates strictly in RAS space instead of the original space, leading to a spatial misalignment.

**Update (July 21, 2026)**:
Ran evaluation on a single validation scan (`train_18379_a_1`, Large finding of 109,373 voxels) using `tile_step_size = 0.5`. The result yielded a Dice of **0.4289** (42.89%), completely resolving the anomaly and proving the `0.75` aggressive overlap was the root cause of the degradation.

**Next Action Recommended**: Rerun full validation set with `tile_step_size = 0.5` to establish the definitive VoxTell zero-shot baseline, then proceed with extracting qualitative worst-performing scans.
